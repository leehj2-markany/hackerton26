# 비개발자 해커톤 사전 제출 과제 양식

**팀명:** Anybridge (가제)

## 1. 프로젝트 소개(300자 내외)

**Anybridge**는 마크애니 고객 지원을 위한 AI 챗봇 솔루션입니다. 웹사이트에 방문한 고객이 질문을 입력하면, Gemini 2.5 Pro AI가 내부 기술 자료(PDF, DOC)를 실시간으로 검색하여 정확한 답변을 제공합니다. 이를 통해 단순 반복 문의를 자동화하여 업무 효율을 높이고, 고객은 24시간 대기 없이 즉각적인 답변을 받을 수 있습니다. AI가 해결하기 어려운 복잡한 문제는 담당자에게 원활하게 연결(에스컬레이션)하여 기술 지원의 연속성을 보장하는 것을 목표로 합니다.

- **애플리케이션 입/출력 흐름:**
  - **입력:** 고객의 회사명, 자연어 질문
  - **처리:**
    1. 고객 정보 조회 (Google Sheets)
    2. 질문 의도 파악 및 내부 문서 검색 (Gemini 2.5 Pro File Search)
    3. AI 답변 및 근거 자료 생성
    4. 필요시 Slack으로 담당자에게 에스컬레이션
  - **출력:** AI 답변, 관련 근거 문서, 담당자 연결 상태 실시간 표시

## 2. 애플리케이션의 입력

애플리케이션이 입력받는 핵심 데이터는 다음과 같습니다.

1.  **고객사 이름 (텍스트)**
    - 예시: `SK하이닉스`
    - 고객을 식별하고, 과거 유지보수 이력 및 사용 중인 제품 정보를 조회하는 데 사용됩니다.

2.  **사용자 질문 (텍스트)**
    - 예시: `Document SAFER를 v3.2로 업그레이드하면 성능이 개선되나요?`
    - AI가 답변을 생성하기 위한 핵심 입력입니다. 대화의 맥락을 포함한 전체 대화 내용이 함께 전달되어 보다 정확한 답변을 생성합니다.

## 3. 애플리케이션의 동작 과정

사용자 문의부터 해결까지의 과정은 다음과 같이 진행됩니다.

1.  **고객 식별:** 사용자가 "SK하이닉스"와 같이 회사명을 입력하면, 백엔드는 유지보수 현황이 정리된 Google Sheets를 조회하여 해당 고객의 정보(사용 제품, 버전, 담당자 등)를 화면에 표시합니다.

2.  **질문 접수 및 AI 분석:** 사용자가 질문을 입력하면, 프론트엔드는 백엔드의 `/api/chat`을 호출합니다.

3.  **AI 답변 생성 (Gemini 2.5 Flash):** 백엔드 서버는 Gemini 2.5 Flash의 RAG 파이프라인을 활용하여 사전에 벡터화된 기술 문서, 매뉴얼 등에서 질문과 관련된 정보를 시맨틱 검색합니다. 질문 사전 분석(analyzeQuestion) 결과를 답변 생성에 재활용하여 LLM 중복 호출을 방지하고, 단순 질문은 안전 검증 시 Claude LLM 호출을 스킵하여 응답 속도를 최적화합니다. 시스템 프롬프트에는 기능 중복 제품 구분 안내 규칙이 포함되어, 고객이 특정 기능(워터마크, 암호화 등)을 언급할 때 해당 기능이 여러 제품에 걸쳐 제공되는 경우 적용 대상별로 구분하여 안내합니다. AI는 검색된 내용을 바탕으로 답변과 함께 어떤 문서를 참조했는지 근거를 생성하여 프론트엔드에 전달합니다.

4.  **답변 및 '생각 과정' 시각화:** 프론트엔드는 AI가 생성한 답변과 함께, AI가 어떤 단계를 거쳐 답변을 만들었는지 '생각 과정(Thinking Process)'을 시각적으로 보여주어 답변에 대한 신뢰도를 높입니다.

5.  **에스컬레이션:** AI가 답변하기 어렵거나, 사용자가 "담당자 연결"을 요청하면 에스컬레이션 프로세스가 시작됩니다.
    - 백엔드의 `/api/escalate` API가 호출되어 관련 담당자들이 포함된 Slack 채널을 자동으로 생성합니다.
    - 프론트엔드에서는 어떤 담당자들이 채널에 참여했는지 실시간으로 현황을 보여줍니다. (예: "2/4명 참여 완료")

## 4. 기대하는 애플리케이션의 출력

1.  **AI 답변:** 사용자의 질문에 대한 구체적인 텍스트 답변입니다.
    - 예시: "네, v3.2 업그레이드 시 OOO 기능이 추가되어 성능 개선이 기대됩니다. 관련 내용은 'Document SAFER v3.2 릴리즈 노트.pdf' 5페이지를 참고해 주세요."

2.  **참조 문서 목록:** AI가 답변을 생성하는 데 사용한 내부 문서 목록입니다. 사용자는 직접 문서를 확인하여 더 깊이 있는 정보를 얻을 수 있습니다.

3.  **담당자 연결 상태:** 에스컬레이션 시, Slack 채널에 초대된 담당자들의 참여 현황을 실시간으로 보여주어 고객이 대기 시간을 예측할 수 있게 합니다.

4.  **관리자 대시보드:** AI의 답변 해결률, 평균 응답 시간, 가장 많이 질문받은 제품군 등 고객 지원 현황을 한눈에 파악할 수 있는 KPI 데이터를 제공합니다.
