# Anybridge 챗봇 UI/UX 리뷰

> 리뷰 일시: 2026-03-12  
> 대상: 챗봇 위젯 (Chatbot.jsx + 관련 컴포넌트)  
> 기준: 실제 배포 화면 스크린샷 + 코드 분석

---

## 🟢 잘 된 점

| 항목 | 설명 |
|------|------|
| 헤더 디자인 | 그라데이션 + 온라인 상태 표시가 신뢰감을 줌 |
| 메시지 구분 | AI/사용자/에이전트/시스템/Slack 메시지 타입별 시각 구분 명확 |
| 마크다운 렌더링 | 제품명 볼드, 리스트, 링크 등 구조화된 답변 표시 |

## 🔴 개선 필요 사항

### P1. AI 답변 버블이 너무 길다
- **현상**: AI 답변이 화면의 60% 이상 차지. 모바일에서 스크롤 과다.
- **원인**: 답변 길이 제한 없이 전체 텍스트를 한 버블에 렌더링
- **제안**: 3~4문장 이후 "더보기" 접기 패턴 적용. 펼치면 전체 표시.
- **파일**: `Chatbot.jsx` — AI 메시지 렌더링 부분
- **난이도**: 중
- **효과**: 높음 (모바일 UX 대폭 개선)

### P2. 입력창 영역 시각적 분리 부족
- **현상**: 메시지 영역과 입력창의 경계가 불명확. 입력창이 메시지에 묻힘.
- **원인**: 입력 영역에 상단 border/shadow 없음
- **제안**: 입력 영역에 `border-t border-gray-200 shadow-inner` 또는 약간의 배경색 차이 적용
- **파일**: `Chatbot.jsx` — 입력 영역 컨테이너
- **난이도**: 낮음
- **효과**: 높음 (즉시 체감)

### P3. 타임스탬프 미표시
- **현상**: 메시지에 시간 표시 없음. 에스컬레이션 후 "언제 보낸 건지" 알 수 없음.
- **원인**: timestamp 데이터는 있으나 렌더링하지 않음
- **제안**: 메시지 버블 하단에 `오후 2:30` 형식으로 작게 표시. 연속 메시지는 마지막만.
- **파일**: `Chatbot.jsx` — 메시지 렌더링 부분
- **난이도**: 낮음
- **효과**: 중간 (대화 맥락 파악에 도움)

### P4. 사용자 메시지 버블 줄간격
- **현상**: 파란 배경 사용자 버블에서 긴 문장의 줄간격이 좁아 가독성 저하
- **원인**: `leading` 클래스 미적용
- **제안**: 사용자 메시지 텍스트에 `leading-relaxed` 추가
- **파일**: `Chatbot.jsx` — 사용자 메시지 렌더링
- **난이도**: 낮음
- **효과**: 중간

### P5. 제품명 시각적 강조 부족
- **현상**: "Screen SAFER" 등 제품명이 볼드로만 표시. 긴 답변에서 묻힘.
- **원인**: 마크다운 `**bold**`만 사용, 별도 스타일링 없음
- **제안**: 제품명에 pill 배지 스타일 적용 (파란 배경 + 흰 텍스트) 또는 마크다운 커스텀 컴포넌트에서 특정 키워드 감지 후 배지 렌더링
- **파일**: `Chatbot.jsx` — `markdownComponents.strong`
- **난이도**: 중
- **효과**: 중간 (시각적 품질 향상)

### P6. 전제조건/주의사항이 본문에 묻힘
- **현상**: "에이전트 설치와 사용자 정보 DB 구성이 전제됩니다" 같은 중요 정보가 일반 텍스트로 흘러감
- **원인**: AI 답변이 plain text로만 구성, 구조화된 안내 박스 없음
- **제안**: 프롬프트에서 전제조건을 `> ⚠️ ...` 블록쿼트로 출력하도록 유도하고, 마크다운 blockquote 스타일을 경고 박스(노란 배경)로 렌더링
- **파일**: `geminiClient.js` (프롬프트) + `Chatbot.jsx` (마크다운 컴포넌트) + `index.css`
- **난이도**: 중
- **효과**: 높음 (고객이 중요 정보를 놓치지 않음)

### P7. 모바일 키보드 대응
- **현상**: 모바일 전체화면 모드에서 키보드가 올라오면 입력창이 가려질 수 있음
- **원인**: `h-full` 고정 높이 + 키보드 높이 미고려
- **제안**: `dvh` (dynamic viewport height) 사용 또는 `visualViewport` API로 키보드 높이 감지
- **파일**: `Chatbot.jsx` — 챗봇 윈도우 컨테이너
- **난이도**: 중
- **효과**: 중간 (모바일 사용자 한정)

### P8. ThinkingPanel 닫기 후 재확인 불가
- **현상**: ThinkingPanel이 답변 생성 후 사라지면 AI 사고 과정을 다시 볼 수 없음
- **원인**: `showThinking`이 false로 전환되면 ThinkingPanel 언마운트
- **제안**: AI 메시지에 "🧠 분석 과정 보기" 토글 버튼 추가. 클릭하면 해당 메시지의 thinkingProcess 표시.
- **파일**: `Chatbot.jsx` — AI 메시지 렌더링 + 상태 관리
- **난이도**: 중
- **효과**: 중간 (투명성/신뢰도 향상, 해커톤 데모에서 어필 포인트)

### P9. Quick Reply 버튼 스크롤 힌트 없음
- **현상**: 에스컬레이션 후 Quick Reply 버튼이 가로 스크롤인데 스크롤 가능하다는 시각적 힌트 없음
- **원인**: `quick-reply-scroll` CSS에 스크롤바 숨김 처리만 있음
- **제안**: 오른쪽 끝에 페이드 그라데이션 오버레이 추가하여 "더 있음" 암시
- **파일**: `Chatbot.jsx` + `index.css`
- **난이도**: 낮음
- **효과**: 낮음

### P10. 에러 상태 UX 미흡
- **현상**: API 실패 시 "일시적인 문제가 발생했습니다" 텍스트만 표시. 재시도 버튼 없음.
- **원인**: catch 블록에서 에러 메시지만 추가
- **제안**: "다시 시도" 버튼 추가. 클릭 시 마지막 사용자 메시지로 재요청.
- **파일**: `Chatbot.jsx` — `handleAIResponse` catch 블록
- **난이도**: 낮음
- **효과**: 중간 (사용자 이탈 방지)

### P11. 신뢰도 표시 미활용
- **현상**: 백엔드에서 confidence/confidenceScore를 반환하지만 UI에 표시하지 않음
- **원인**: 메시지 객체에 confidence 저장은 하지만 렌더링 코드 없음
- **제안**: AI 메시지 하단에 신뢰도 배지 표시 (🟢 높음 / 🟡 중간 / 🔴 낮음). 낮을 때는 "정확하지 않을 수 있습니다" 안내.
- **파일**: `Chatbot.jsx` — AI 메시지 렌더링
- **난이도**: 낮음
- **효과**: 높음 (해커톤 심사에서 AI 투명성 어필)

### P12. Slack 메시지 버블 디자인
- **현상**: Slack 전달 메시지가 일반 텍스트로 표시. Slack 느낌이 없음.
- **원인**: `type: 'slack'` 메시지에 별도 스타일 없음 (코드에서 확인 필요)
- **제안**: Slack 아이콘 + 채널명 헤더 + 코드블록 스타일로 Slack 메시지 느낌 연출
- **파일**: `Chatbot.jsx` — slack 타입 메시지 렌더링
- **난이도**: 중
- **효과**: 중간 (Slack 연동 데모 시 시각적 임팩트)

### P13. 접근성 (a11y) 기본 사항 미충족
- **현상**: 버튼에 aria-label 없음, 키보드 네비게이션 미지원, 색상 대비 미검증
- **원인**: 접근성 고려 없이 구현
- **제안**: 최소한 주요 버튼(전송, 닫기, 최소화)에 aria-label 추가. Enter 키 전송은 이미 구현됨.
- **파일**: `Chatbot.jsx` 전체
- **난이도**: 낮음
- **효과**: 낮음 (해커톤에서는 우선순위 낮지만 언급 가치 있음)
