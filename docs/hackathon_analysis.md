- `squad-hotline-dlp` - DLP(DST) 질문 (워크플로우 "DST 질문 생성 수집" 존재)
- `squad-hotline-tracer` - Tracer 질문 (워크플로우 "Tracer 질문 생성 수집" 존재)
- `squad-hotline-ait-e-page` - AIT(e-Page SAFER) 질문 (워크플로우 존재)

**패턴**: 제품별로 분산된 기술지원 채널 → 반복적 Q&A, 지식 분산 문제

**패턴**: 프로젝트 정보가 채널별로 사일로화 → 유사 프로젝트 간 지식 공유 부재

## 🤖 AI로 해결 가능한 문제점

### 문제 1: 기술지원 Q&A 반복 (가장 큰 기회)
- **현상**: DRM, DLP, Tracer, e-Page, SafePC 등 제품별 hotline 채널에서 유사한 질문이 반복
- **증거**: 이미 Slack 워크플로우("질문 생성 수집")를 만들어 질문을 구조화하려는 시도가 있음
- **영향**: 개발자/기술지원 인력의 시간 낭비, 응답 품질 불균일

### 문제 2: 프로젝트 지식 사일로
- **현상**: 50개 이상의 프로젝트 채널에 구축 경험, 이슈 해결 방법이 분산
- **증거**: 유사 프로젝트(영상반출, 패스워드, 선별관제 등)가 고객사별로 반복
- **영향**: 신규 프로젝트 시작 시 과거 경험 활용 불가

### 문제 3: 영업/제안 프로세스 비효율
- **현상**: 제안서 작성, 고객 문의 이관, 유지보수 관리가 수동
- **증거**: `0-markany-csteam`에서 수동 멘션으로 이관, 유지보수 시트 별도 관리
- **영향**: 영업 기회 누락, 대응 속도 저하

### 문제 4: 파트너/협력사 커뮤니케이션
- **현상**: 파트너별 채널에서 제품 자료 공유, 기술 문의 반복
- **증거**: 각 파트너 채널에 "제품 정보 및 자료" 폴더가 반복 생성
- **영향**: 파트너 온보딩 시간 증가, 동일 자료 반복 공유

## 💡 해커톤 제안 아이디어 (2일 구현 가능)

### 🥇 아이디어 1: "마크애니 기술지원 AI 챗봇" (최우선 추천)
- **문제**: 제품별 hotline 채널에서 반복되는 기술 Q&A
- **솔루션**: 
  - 기존 hotline 채널의 질문/답변 데이터를 RAG(Retrieval-Augmented Generation)로 학습
  - Slack 봇으로 자동 답변 또는 유사 질문/답변 추천
  - 답변 불가 시 적절한 담당자에게 자동 라우팅
- **기존 데이터**: Slack 워크플로우로 이미 구조화된 Q&A 데이터 활용 가능
- **기술 스택**: Slack Bot API + OpenAI/Claude API + Vector DB (Pinecone/ChromaDB)
- **2일 MVP**: 1개 제품(예: DRM) hotline 데이터로 프로토타입

### 🥈 아이디어 2: "프로젝트 인텔리전스 봇"
- **문제**: 유사 프로젝트 간 지식 공유 부재
- **솔루션**:
  - 프로젝트 채널의 대화/문서를 분석하여 프로젝트 요약 자동 생성
  - 신규 프로젝트 시작 시 유사 과거 프로젝트 자동 추천
  - "영상반출 구축 시 주의사항" 같은 질문에 과거 프로젝트 경험 기반 답변
- **기존 데이터**: 50+ 프로젝트 채널의 대화 히스토리
- **기술 스택**: Slack API + LLM + Embedding Search

### 🥉 아이디어 3: "고객 문의 자동 분류 & 라우팅"
- **문제**: `0-markany-csteam`에서 수동 멘션으로 문의 이관
- **솔루션**:
  - 고객 문의 내용을 AI가 자동 분류 (제품별, 긴급도별)
  - 적절한 팀/담당자에게 자동 라우팅
  - 유사 과거 문의 및 해결 방법 자동 첨부
- **기존 데이터**: CS팀 채널 히스토리, hotline 채널 데이터
- **기술 스택**: Slack Bot + NLP 분류 모델 + 라우팅 로직

### 아이디어 4: "미팅 녹취 → 액션 아이템 자동 추출"
- **문제**: `squad-미팅녹취모음` 채널 존재 → 녹취는 하지만 활용이 부족
- **솔루션**: 녹취록에서 액션 아이템, 결정 사항, 담당자 자동 추출
- **기술 스택**: Whisper API + LLM 요약

### 아이디어 5: "제안서 자동 초안 생성기"
- **문제**: 고객사별 제안서 반복 작성
- **솔루션**: 과거 제안서 + 고객 요구사항 기반 초안 자동 생성
- **기존 데이터**: 제안서 관련 채널의 문서들
