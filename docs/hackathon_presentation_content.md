# ANY 브릿지 — AI와 우리 사이의 거리를 0으로 만든다
## 마크애니 AI 해커톤 2026 | 팀 미생:코딩편

### 프로젝트 정보
- 프로젝트명: ANY 브릿지 (ANYbridge)
- 팀명: 미생:코딩편
- 설명: AI와 우리 사이의 거리를 0으로 만든다
- 컨셉: AI 1차 필터링 → 인간이 붙는다 → 지식이 쌓인다

### 팀 구성
- 이태영 본부장 — 총괄, 아이디어, 발표 지원
- 송인찬 팀장 — 기술 검증, 데모 시나리오, 라이브 데모
- 이현진 팀장 — AI 구현, 통합, 발표
- 이병우 상무 — 비즈니스 검증, 실사용 관점 피드백

## 핵심 아이디어 (회의 확정)

### 3단계 구조: AI와 인간의 조합

① 1차 — AI 챗봇 (홈페이지)
   - 고객명 입력 → 유지보수 현황 시트 기반 담당자 자동 매칭
   - 정책기능서 학습 RAG 기반 자연어 질의응답
   - "윈 10에서 윈 11 업그레이드 가능한가요?" → AI 즉시 답변

② 2차 — 담당자 연결 (카톡방 자동 개설)
   - AI로 해결 안 되면 → "카톡방 개설" 버튼
   - 영업 + 엔지니어 + 유지보수 담당자 자동 초대
   - 실시간 그룹 채팅으로 문제 해결
   - 기술팀 공식 답변 → 자동으로 고객에게 전달

③ 3차 — 완료 & 축적
   - 해결 완료 → 만족도 평가 → 방 자동 해제
   - 모든 대화 이력 DB 축적 → AI 학습 데이터로 고도화
   - 구글 시트 대시보드: 제품별 문의 건수, 답변 완료율, 응답 시간

### Use Case (데모용)
1. SK하이닉스 — 챗봇 → 카톡방 개설 → 해결 → 방 해제 풀 시나리오
2. 국민건강보험공단 — 챗봇 1차 해결 완료 시나리오
3. 국방부 (담당자: 송인찬) — 커스터마이징 문의 → 기술팀 에스컬레이션 시나리오

### 데이터 소스
- 박희수 본부장 정제 문서 (정책기능서) → RAG 학습 (100건+ 테스트 완료)
- 세실님 유지보수 현황 시트 → 고객-담당자 매핑
- 마크애니 홈페이지 소스 → 복제 가능

## 문제 정의
1. 인바운드 리드 응답 시간이 너무 느림
2. 기고객 매출 볼륨/비중이 큰데 관리가 안 됨 → 기고객 만족도 최대화 필요
3. MTG↔MCG 기술 자산 파편화 → 조직 자산으로 축적 안 됨

## 발표 스토리라인 (회의에서 논의)
- "Customer First" 한 장 넣기
- 작년 해커톤 1등(캘린더 자동 회의실 잡기) → 지금 쓰고 있나요? → 우리는 실적용 가능
- 개발자 팀들: 완성도 높지만 아이디어 부실, 내부 편의 중심
- 우리: 고객 접점 + 내부 접점 둘 다 건드림
- AI와 인간의 조합 강조 — "AI가 우리의 문을 열어준다"
- 심사위원(금동기, 임병국) 관점: 결과 중심, 실사용 가능성이 핵심
- 비개발자의 감성 터치 — 코딩이 아니라 아이디어와 스토리로 승부

## 주의사항 (회의에서 논의)
- 구현 가능한 수준에서 해야 함 — 아이디어 좋아도 구현 안 되면 꽝 (작년 사례)
- 최대한 심플하게 — 전선 넓히면 꼬임
- 자충수 만들 일 하면 안 됨 (유지보수 견적 자동화 같은 편협한 주제 피하기)
- 설명은 은유적으로 — 다른 팀에 안 들키게
- Salesforce 기능 활용 언급하면 심사위원에게 먹힘
- 코드 적게 쓰는 방향 — 결과를 보는 사람들이니까 과정은 별로 안 중요
