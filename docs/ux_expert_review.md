## 1라운드: 전체 구조 평가

### 돈 노먼 (사용자 중심 디자인)

> 전체적으로 "시스템 상태의 가시성(Visibility of System Status)" 원칙을 잘 따르고 있습니다. AI 사고 과정 패널, 담당자 연결 상태, 처리 중 인디케이터 등이 사용자에게 현재 무슨 일이 일어나고 있는지 명확하게 전달합니다.

**장점:**
- AI의 사고 과정을 투명하게 보여주는 것은 신뢰 구축에 매우 효과적
- 담당자 입장 순서를 실시간으로 보여주는 것은 "대기 불안"을 줄여줌
- 슬랙 연동 메시지가 "이 시스템이 실제로 뒤에서 일하고 있다"는 느낌을 줌

**문제점:**
- 챗봇 창이 450x700px 고정인데, 메시지가 많아지면 스크롤이 과도해짐. 사용자가 이전 맥락을 잃기 쉬움
- 에스컬레이션 후 메시지 타입이 4가지(ai, agent, slack, system)로 늘어나는데, 시각적 구분이 충분한지 재검토 필요
- "만족도 확인 → 불만족 → 에스컬레이션" 경로가 3단계인데, 2단계로 줄일 수 있음

**개선안:**
1. 메시지 영역에 "날짜/시간 구분선" 추가하여 대화 흐름 파악 용이하게
2. 에스컬레이션 후에는 챗봇 창 크기를 자동으로 넓히거나, 전체 화면 모드 제공
3. "불만족" 클릭 시 바로 "담당자 연결하기" 버튼을 보여주지 말고, "어떤 부분이 부족했나요?" 선택지를 먼저 제공 → 이 데이터가 담당자에게 전달되면 더 정확한 답변 가능

---

### 제이콥 닐슨 (사용성 휴리스틱)

> 닐슨 10가지 휴리스틱 기준으로 평가하겠습니다.

| 휴리스틱 | 점수 | 평가 |
|----------|------|------|
| 1. 시스템 상태 가시성 | ★★★★☆ | AI 사고 과정, 담당자 연결 상태 우수. 답변 대기 시 남은 시간 표시 없음 |
| 2. 시스템과 현실 세계 일치 | ★★★★★ | 슬랙 메시지, 담당자 카드 등 실제 업무 환경과 일치 |
| 3. 사용자 제어와 자유 | ★★★☆☆ | 에스컬레이션 취소 불가, 대화 초기화 기능 없음 |
| 4. 일관성과 표준 | ★★★★☆ | 메시지 타입별 색상 구분 일관적. 버튼 스타일 통일 |
| 5. 오류 예방 | ★★★☆☆ | 빈 메시지 전송 방지됨. 처리 중 입력 비활성화 좋음 |
| 6. 인식 > 회상 | ★★★★☆ | 고객 정보 패널이 항상 보임. 참조 문서 링크 제공 |
| 7. 유연성과 효율성 | ★★☆☆☆ | 키보드 단축키 없음, 빠른 질문 템플릿 없음 |
| 8. 미학적 최소 디자인 | ★★★☆☆ | 메시지 타입이 많아 시각적 복잡도 높음 |
| 9. 오류 인식/진단/복구 | ★★☆☆☆ | 네트워크 오류, API 실패 시 처리 없음 |
| 10. 도움말과 문서 | ★★☆☆☆ | 사용법 안내 없음, 첫 사용자 온보딩 없음 |

**핵심 개선안:**
1. **빠른 질문 버튼(Quick Replies)**: 에스컬레이션 후 자주 묻는 질문을 칩 형태로 제공
2. **답변 대기 프로그레스**: "담당자가 답변을 준비하고 있습니다..." 대신 예상 대기 시간 또는 프로그레스 바 표시
3. **대화 내보내기**: 상담 종료 후 대화 내용을 PDF/이메일로 내보내기 기능
4. **에스컬레이션 취소**: "아, 괜찮아요. AI 답변으로 충분합니다" 옵션 제공

## 2라운드: 메시징 UX 심층 분석

### 앨리스 리 (Slack Design VP)

> 슬랙 메시지 연동 UI가 인상적입니다. 실제 슬랙에서 영감을 받은 것이 보이는데, 몇 가지 개선하면 더 자연스러워질 수 있습니다.

**슬랙 메시지 UI 개선:**

1. **채널명 표시 개선**: 현재 `#국방부-DRM-구축`이 메시지 본문 안에 있는데, 헤더 영역으로 분리하는 게 좋음
```
  💬 Slack  #국방부-DRM-구축
  ┃ 채소희_봇  오후 3:15
  ┃ @송인찬 님, 고객님께서 [구축 가능성]
  ┃ 관련 문의를 주셨습니다.
```
→ 발신자(채소희_봇)와 시간을 추가하면 실제 슬랙 메시지처럼 보임

2. **스레드 표시**: 슬랙에서는 답변이 스레드로 달리는데, 담당자 답변도 "↳ 스레드에서 답변" 형태로 표시하면 슬랙 연동이 더 실감남

3. **리액션 이모지**: 슬랙에서 메시지에 👀(확인 중), ✅(완료) 리액션을 다는 것처럼, 담당자가 메시지를 확인했을 때 👀 이모지가 붙고, 답변 완료 시 ✅로 바뀌는 애니메이션

**담당자 메시지 개선:**

4. **타이핑 인디케이터**: 15초(현재 5초) 대기 중 "송인찬님이 입력 중..." 같은 타이핑 인디케이터를 보여주면 사람이 실제로 타이핑하는 느낌

5. **메시지 그룹핑**: 같은 담당자의 연속 메시지는 아바타를 한 번만 표시하고 메시지만 이어붙이기 (슬랙 스타일)

### 김민수 교수 (고객 만족도)

> B2B 고객 지원 관점에서 이 시스템의 가장 큰 강점은 "고객이 방치되지 않는다는 느낌"을 주는 것입니다. 하지만 몇 가지 고객 심리 관점의 개선이 필요합니다.

**고객 심리 분석:**

1. **대기 시간 인지 관리 (Perceived Wait Time)**
   - 현재: "담당자가 답변을 준비하고 있습니다..." (pulse 애니메이션)
   - 문제: 고객은 "얼마나 기다려야 하는지" 모르면 실제보다 길게 느낌
   - 개선: "송인찬님이 답변을 작성하고 있습니다 (예상 약 1분)" 또는 진행 단계 표시
   - 연구 근거: Maister(1985)의 대기 심리학 - "설명된 대기는 설명되지 않은 대기보다 짧게 느껴진다"

2. **감정 전환 포인트 (Emotional Transition)**
   - AI 답변 불만족 → 에스컬레이션은 고객의 감정이 부정적인 상태
   - 채소희의 첫 인사가 이 감정을 전환하는 핵심 포인트
   - 현재 인사가 좋지만, "불편을 드려 죄송합니다"를 먼저 언급하는 것이 더 효과적
   - 개선: "불편을 드려 죄송합니다 😔 제가 직접 도와드리겠습니다. 관련 전문가분들을 바로 연결해 드릴게요!"

3. **종료 경험 (Closing Experience)**
   - 연락처 카드 제공은 매우 좋음. "이 상담이 끝나도 연결이 끊기지 않는다"는 안심감
   - 추가 제안: "오늘 상담 요약"을 자동 생성하여 함께 제공
   - 상담 만족도 별점(1~5) 수집 → 서비스 개선 데이터로 활용

4. **선택지 프레이밍 (Choice Framing)**
   - 현재: [💬 질문 더하기] [👋 채팅 종료]
   - "채팅 종료"라는 표현이 다소 차가움
   - 개선: [💬 더 물어볼게요] [✅ 충분합니다, 감사해요]
   - 긍정적 프레이밍이 고객 만족도를 높임

## 3라운드: 행동 디자인 & 참여 루프

### 에이미 조 킴 (행동 디자인)

> 이 시스템의 핵심 가치는 "AI → 사람" 전환이 자연스럽다는 것입니다. 하지만 사용자 참여를 더 높일 수 있는 디자인 패턴이 있습니다.

**참여 루프 개선:**

1. **프로그레시브 디스클로저 (Progressive Disclosure)**
   - 현재: 모든 정보가 한꺼번에 표시됨
   - 개선: AI 사고 과정을 기본 접힌 상태로 두고, "AI가 어떻게 생각했는지 보기" 링크로 펼치기
   - 이유: 일반 고객은 답변만 원하고, 기술적 고객은 과정도 보고 싶어함

2. **마이크로 인터랙션 추가**
   - 담당자 입장 시: 카드가 슬라이드인 + 짧은 사운드 효과 (선택적)
   - 슬랙 메시지 전송 시: "전송됨 ✓" → "확인됨 ✓✓" 단계 표시 (WhatsApp 스타일)
   - 답변 완료 시: 부드러운 체크마크 애니메이션

3. **컨텍스트 유지 (Context Persistence)**
   - 10여 차례 질의가 이어지면 이전 맥락을 잊기 쉬움
   - "지금까지 논의한 내용" 요약 카드를 중간중간 삽입
   - 예: 5번째 질문 후 "📋 지금까지 구축 기간, 비용, 연동, 보안, 교육에 대해 논의했습니다"

4. **Quick Reply 칩**
   - 에스컬레이션 후 매번 타이핑하는 대신, 예상 질문을 칩으로 제공
```
  [구축 기간은?] [비용이 궁금해요] [레퍼런스 있나요?] [계약 절차는?]
```
   - 고객이 탭 한 번으로 질문 가능 → 참여율 증가

## 전문가 총평

**돈 노먼**: "AI와 사람의 협업을 투명하게 보여주는 점이 이 시스템의 핵심 차별점입니다. 사용자가 '블랙박스'가 아닌 '유리 상자'를 보는 느낌을 받습니다."

**앨리스 리**: "슬랙 연동 UI가 실제 업무 환경과 자연스럽게 연결되는 점이 B2B 고객에게 신뢰감을 줍니다. 타이핑 인디케이터와 리액션만 추가해도 완성도가 크게 올라갈 것입니다."

**김민수 교수**: "고객 심리 관점에서 '방치되지 않는다'는 느낌이 가장 중요한데, 채소희 봇의 중간 안내가 이를 잘 해결하고 있습니다. 종료 경험의 연락처 카드는 B2B에서 매우 효과적인 패턴입니다."

**제이콥 닐슨**: "사용성 기본기는 잘 갖춰져 있습니다. Quick Reply와 에스컬레이션 취소 기능을 추가하면 사용성 점수가 크게 향상될 것입니다."

**에이미 조 킴**: "10여 차례 질의가 이어지는 긴 세션에서 사용자 참여를 유지하려면 Quick Reply 칩과 중간 요약이 핵심입니다."
