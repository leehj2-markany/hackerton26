# ANY 브릿지 — AI 성과 대시보드 PRD & 구현 내역

## 1. 개요

ANY 브릿지 AI 성과 대시보드는 AI 챗봇의 프리세일즈 지원 성과를 실시간으로 시각화하는 관리자용 화면이다.
Salesforce Agentforce Command Center 패턴을 레퍼런스로 설계했으며, AI 자동 처리와 휴먼 개입을 명확히 구분하여 보여주는 것이 핵심이다.

- 배포 URL: https://hackerton-kappa.vercel.app (라우트: `/dashboard`)
- 데이터: 가상 데이터 500건 기반 (시뮬레이션)
- 기술 스택: React 18 + Vite 5, Tailwind CSS, 라이트 테마

## 2. 요구사항 (PRD)

### 2.1 핵심 목표
| # | 요구사항 | 우선순위 |
|---|---------|---------|
| R1 | AI 자동 해결률, 응답 시간, 고객 만족도, 비용 절감 등 핵심 KPI를 한눈에 파악 | P0 |
| R2 | AI가 처리한 케이스와 휴먼이 개입한 케이스를 명확히 구분하여 시각화 | P0 |
| R3 | 월별 AI vs 휴먼 처리 추이를 통해 AI 도입 효과 트렌드 확인 | P1 |
| R4 | 제품별 AI 해결률 차이를 파악하여 지식 베이스 보강 포인트 식별 | P1 |
| R5 | AI 에이전트(RAG, Safety, Escalation) 각각의 실시간 상태 모니터링 | P1 |
| R6 | 실시간 이벤트 로그로 시스템 동작 현황 파악 | P2 |

### 2.2 디자인 원칙
- Salesforce Agentforce Command Center 스타일의 라이트 테마
- 정보 밀도 높되 시각적 피로 최소화 (white card + subtle border)
- AI(파란색 🤖) vs 휴먼(주황색 👤) 컬러 코딩 일관 적용
- 외부 차트 라이브러리 없이 CSS 기반 바 차트로 경량 구현

### 2.3 제외 항목
- Before/After 비교 섹션 (v1에서 제거 — KPI 카드의 Before 값으로 대체)
- 실시간 API 연동 (현재 가상 데이터, 향후 `/api/dashboard/stats` 연결 예정)

## 4. 구현 내역

### 4.1 파일 구조

| 파일 | 역할 |
|------|------|
| `frontend/src/components/Dashboard.jsx` | 대시보드 메인 컴포넌트 (7개 서브 컴포넌트 포함) |
| `frontend/src/data/dashboardData.js` | 가상 데이터 (KPI, 케이스 이력, 월별/제품별 통계, 에이전트, 이벤트) |

### 4.2 컴포넌트 구성

| 컴포넌트 | 설명 |
|----------|------|
| `Dashboard` | 메인 레이아웃. 헤더(sticky) + 5개 Row + 푸터. 실시간 시계, 필터 상태 관리 |
| `HeroKPICard` | KPI 카드 1개. 아이콘, 값, 단위, 변화량 배지, Before 값 표시 |
| `CaseHistoryTable` | AI vs 휴먼 처리 이력 테이블. `useMemo`로 필터링, 스크롤 가능(max-h-80) |
| `MonthlyChart` | 월별 AI/휴먼/진행중 CSS 스택 바 차트. 범례 포함 |
| `ProductDistribution` | 제품별 AI vs 휴먼 프로그레스 바. 4개 제품(DRM, Document SAFER, Print SAFER, Screen SAFER) |
| `AgentStatusCards` | AI 에이전트 3개(RAG, Safety, Escalation) 상태 카드. 정확도/응답시간/오늘 처리건 |
| `EventLog` | 실시간 이벤트 로그. 타입별 컬러 코딩(AI=파랑, 휴먼=주황, Safety=보라) |

### 4.3 데이터 구조 (`dashboardData.js`)

| Export | 타입 | 설명 |
|--------|------|------|
| `heroKPIs` | Array(4) | AI 자동 해결률, 평균 응답, 고객 만족도, 건당 비용 절감 |
| `caseHistory` | Array(20) | 최근 케이스 20건. `resolvedBy: 'ai' \| 'human'`으로 구분 |
| `monthlyAIvsHuman` | Array(3) | 12월~2월 월별 AI/휴먼/진행중 건수 + AI 비율 |
| `productAIvsHuman` | Array(4) | 제품별 총건수, AI건수, 휴먼건수, AI 비율 |
| `aiAgents` | Array(3) | RAG 검색 엔진, AI 안전성 가드, 에스컬레이션 매니저 |
| `recentEvents` | Array(8) | 실시간 이벤트 로그 (ai/human/safety 타입) |

### 4.4 주요 UX 패턴

- 필터 탭: 전체 / 🤖 AI / 👤 휴먼 — 세그먼트 컨트롤 스타일
- 컬러 코딩: AI 처리 = `blue-500` / 휴먼 개입 = `amber-400` / 진행 중 = `gray-300`
- 응답 시간 단위 자동 변환: AI는 "초", 휴먼은 "분" 단위로 표시
- 에이전트 상태: `animate-pulse` 녹색 점으로 활성 상태 표현
- 이벤트 로그: 타입별 배경색 구분 (blue/amber/purple)

### 4.5 디자인 토큰

| 요소 | 스타일 |
|------|--------|
| 카드 | `bg-white rounded-2xl shadow-sm border border-gray-100` |
| 섹션 제목 | `text-sm font-semibold text-gray-500 uppercase tracking-wider` |
| AI 배지 | `bg-blue-50 text-blue-700 rounded-full` |
| 휴먼 배지 | `bg-amber-50 text-amber-700 rounded-full` |
| 긍정 변화 | `bg-emerald-50 text-emerald-600` |
| 헤더 | `bg-white border-b sticky top-0 z-10` |
