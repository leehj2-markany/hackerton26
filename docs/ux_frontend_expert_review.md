## 2. 토론: 잘 된 점 (Strengths)

**UX**: 와이어프레임에서 설계한 에스컬레이션 플로우가 코드에 거의 그대로 반영됐어. 특히 채소희가 질문을 분해하고 → 슬랙 채널에 요청 보내고 → 담당자가 순차 답변하는 3단계 패턴이 `handleFollowUpQuestion()`에 정확히 구현돼 있어. "더 물어볼게요 / 충분합니다" 선택지도 와이어프레임 그대로야.

**FE**: 동의해. 코드 구조도 나쁘지 않아. 메시지 타입을 `ai`, `user`, `agent`, `system`, `slack`, `contactCard`로 분리한 건 확장성 면에서 좋은 설계야. 슬랙 메시지의 `@태그` 파싱해서 파란색 하이라이트 처리한 것도 디테일이 살아있고.

**UX**: 타이핑 인디케이터도 좋아. `typingAgent` 상태로 "송인찬님이 입력 중..." + bounce 애니메이션 보여주는 건 실제 사람이 타이핑하는 느낌을 잘 살렸어. 처리 중일 때 입력창 비활성화 + placeholder 변경도 와이어프레임 타이밍 설계를 잘 따랐어.

**FE**: 대시보드(Dashboard.jsx)는 와이어프레임에 없던 추가 구현인데, 서브에이전트 5개 상태 + KPI + Before/After 비교 + 이벤트 로그까지 꽤 완성도 높게 들어갔어. 해커톤 데모에서 임팩트 있을 거야.

## 3. 토론: 개선이 필요한 점 (Issues)

### 🔴 Critical (데모 전 반드시 수정)

**UX**: ThinkingPanel이 기본적으로 접혀 있어. 와이어프레임에서는 AI 사고 과정이 자동으로 펼쳐지면서 단계별로 보여야 하는데, 현재는 사용자가 클릭해야 펼쳐져. 데모에서 이걸 놓치면 AI 투명성 시연이 안 돼.

**FE**: 맞아. `ThinkingPanel`에서 `expanded` 초기값이 `false`야. 이걸 `true`로 바꾸거나, steps가 추가될 때 자동으로 펼치는 로직이 필요해.

```jsx
// 현재: const [expanded, setExpanded] = useState(false)
// 수정: const [expanded, setExpanded] = useState(true)
```

---

**UX**: 챗봇 창 크기가 `w-[450px] h-[700px]` 고정인데, 와이어프레임의 XL 레이아웃에서는 챗봇이 화면의 70%를 차지하고 정보 패널이 30%를 차지하는 구조야. 현재는 작은 팝업 형태라 에스컬레이션 시 메시지가 길어지면 스크롤이 과도해져.

**FE**: 이건 데모 전략에 따라 다를 수 있어. 팝업 형태가 "실제 고객 지원 챗봇" 느낌을 주긴 하는데, 데모 시연 시에는 전체 화면 모드가 필요할 수 있어. 토글 버튼 하나 추가하면 될 듯.

---

**UX**: `onKeyPress`가 deprecated 경고 나고 있어. `onKeyDown`으로 바꿔야 해.

**FE**: 맞아, React에서 `onKeyPress`는 더 이상 권장하지 않아. 간단한 수정이야.

---

### 🟡 Important (데모 품질 향상)

**UX**: SK하이닉스 시나리오에서 10개 질문 빠른 응답이 와이어프레임에 있는데, 현재 코드에서는 `handleAIResponse`가 항상 같은 패턴(사고 과정 → 답변 → 만족도 확인)을 반복해. 10개 질문을 빠르게 넘기려면 만족도 확인 없이 바로 다음 답변으로 넘어가는 "빠른 모드"가 필요해.

**FE**: `mockScenarios.skhynix.responses`에 10개 답변이 이미 준비돼 있는데, 순차적으로 사용하는 로직이 없어. 응답 인덱스를 추적하는 state가 필요해.

---

**UX**: 정보 패널(InfoPanel)이 `right-[490px]`로 고정 위치인데, 챗봇 크기가 변하면 겹치거나 떨어져. 챗봇과 정보 패널을 하나의 flex 컨테이너로 묶는 게 안정적이야.

**FE**: 동의해. 현재 둘 다 `fixed` 포지션이라 서로 독립적이야. 부모 컨테이너를 만들어서 `flex`로 배치하면 반응형도 자연스러워져.

---

**UX**: 모바일 반응형이 부족해. 와이어프레임에서 모바일은 챗봇이 전체 화면을 차지하고, 정보 패널은 숨기는 구조인데, 현재 `w-[450px]` 고정이라 모바일에서 화면을 벗어나.

**FE**: Tailwind 반응형 클래스를 추가하면 돼. `w-[450px]`을 `w-full sm:w-[450px]`으로, `h-[700px]`을 `h-full sm:h-[700px]`으로 바꾸고, 모바일에서는 `fixed inset-0`으로 전체 화면 처리.

---

**UX**: Quick Reply 칩이 `followUpIndex`에 따라 3개씩 보여주는데, 칩 텍스트가 25자에서 잘려. 모바일에서는 더 짧아야 하고, 데스크톱에서는 전체 텍스트를 보여줘야 해.

**FE**: 반응형 truncate 처리하면 되겠지. 그리고 칩이 3개만 보이는 건 좋은데, "더 보기" 옵션이 있으면 좋겠어.

---

### 🟢 Nice-to-have (시간 여유 시)

**UX**: 메시지 전환 시 애니메이션이 `animate-fadeIn`만 있어. 담당자 메시지는 좌측에서 슬라이드인, 사용자 메시지는 우측에서 슬라이드인 하면 더 자연스러워.

**FE**: CSS 애니메이션 2개만 추가하면 돼. 성능 이슈 없을 거야.

---

**UX**: 신뢰도 배지가 InfoPanel에서는 항상 "🟢 높음 (92%)"으로 고정이야. 와이어프레임에서는 질문마다 신뢰도가 달라져야 하는데.

**FE**: `customerInfo`에 신뢰도를 동적으로 업데이트하는 로직이 없어. AI 응답의 `confidence` 값을 InfoPanel에 전달하는 prop drilling이나 context가 필요해.

---

**UX**: 다크모드 대시보드와 라이트모드 챗봇 사이의 전환이 갑작스러워. 라우팅으로 분리돼 있긴 한데, 네비게이션 UX가 없어.

**FE**: 홈페이지 헤더에 "대시보드" 링크를 추가하거나, 챗봇 내에서 대시보드로 이동하는 버튼이 있으면 좋겠어.

## 4. 접근성 (Accessibility) 체크

| 항목 | 상태 | 비고 |
|------|------|------|
| 키보드 네비게이션 | ⚠️ | 챗봇 내 버튼들에 tabIndex 없음 |
| 스크린 리더 | ⚠️ | aria-label 부재, role 미지정 |
| 색상 대비 | ✅ | 마크애니 블루 #0066CC는 흰색 배경에서 4.5:1 이상 |
| 포커스 표시 | ⚠️ | focus:ring은 입력창에만 있음 |
| 이모지 대체 텍스트 | ❌ | 이모지 아바타에 aria-label 없음 |

**FE**: 해커톤 데모라 접근성 우선순위가 낮긴 하지만, 최소한 `role="dialog"`, `aria-label="챗봇"` 정도는 추가하는 게 좋겠어.

## 5. 코드 품질 체크

| 항목 | 상태 | 비고 |
|------|------|------|
| 미사용 import | ⚠️ | Homepage, InfoPanel, AgentStatus에서 `React` import 불필요 |
| deprecated API | ⚠️ | `onKeyPress` → `onKeyDown` 변경 필요 |
| 미사용 변수 | ⚠️ | `handleAIResponse`의 `question` 파라미터 미사용 |
| 컴포넌트 크기 | ⚠️ | Chatbot.jsx가 ~400줄, 리팩토링 권장 |
| 상태 관리 | ⚠️ | 12개 useState, Context나 useReducer 고려 |
| 에러 처리 | ❌ | try-catch 없음, 에러 바운더리 없음 |
| TypeScript | ❌ | JSX 사용, 타입 안전성 없음 |

**FE**: Chatbot.jsx가 너무 커. 메시지 렌더링 부분을 `MessageBubble` 컴포넌트로 분리하고, 에스컬레이션 로직을 커스텀 훅(`useEscalation`)으로 빼면 가독성이 훨씬 좋아져. 다만 해커톤이라 시간 대비 효과를 따져야 해.

## 7. 우선순위별 수정 권장 사항

### 즉시 수정 (30분 이내)
1. **ThinkingPanel 자동 펼침** — `expanded` 초기값 `true`로 변경
2. **onKeyPress → onKeyDown** — deprecated 경고 제거
3. **미사용 React import 제거** — 3개 파일

### 데모 전 수정 (1~2시간)
4. **모바일 반응형** — 챗봇 전체화면 모드 추가

6. **챗봇 전체화면 토글** — 데모 시연용

### 여유 시 개선
7. InfoPanel 동적 신뢰도 업데이트
8. 메시지 슬라이드 애니메이션
9. Chatbot.jsx 컴포넌트 분리
10. 접근성 기본 속성 추가

## 8. 결론

**UX**: 와이어프레임 대비 구현 완성도가 높아. 에스컬레이션 플로우, 슬랙 연동 UI, 연락처 카드, Quick Reply까지 핵심 시나리오가 전부 동작해. ThinkingPanel 자동 펼침과 모바일 반응형만 잡으면 데모 준비 완료야.

**FE**: 코드 품질은 해커톤 수준에서 충분해. Chatbot.jsx가 좀 크긴 하지만 동작에는 문제없어. 즉시 수정 3건만 반영하면 바로 시연 가능한 상태야. 대시보드까지 포함하면 전체적으로 인상적인 완성도라고 봐.

**공통 의견**: 🟢 **데모 시연 가능 상태 (즉시 수정 3건 반영 후)**
