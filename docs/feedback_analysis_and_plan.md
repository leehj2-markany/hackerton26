# 팀원 피드백 분석 및 설계 업데이트 토론 플랜

## 📋 피드백 요약

### 핵심 피드백
1. **실시간 소통 시각화**: 챗봇에서 담당자들(엔지니어, 영업, 유지보수)이 들어왔음을 표시
2. **대기 메시지**: 고객이 기다리지 않도록 진행 상황 메시지 표시
3. **리얼타임 시나리오**: 문의자와 마크애니 담당자들의 실시간 소통 시연
4. **고객만족도 극대화**: 최상의 고객 경험 제공
5. **가상 담당자 호출**: 
   - 채소희 (고객센터)
   - 송인찬 (세일즈 어카운트)
   - 이현진 (SE 역할)
   - 최소 5명의 서포트 담당자
6. **단계별 응답 시나리오**:
   - 답변 느려지면 → 채소희 1차 안내 메시지
   - 송인찬 2차 답변
   - 이현진 SE로 문제 정의 → 분석 → 답변
7. **에이전트화**: 
   - 질문 쪼개기 및 재정의
   - 담당 롤 정의
   - AI가 롤 지정 (SE, 영업, 담당자)
   - "xx 롤이 답변드릴 예정입니다" 메시지

## 🔍 피드백 분석

### 분석 1: 현재 설계와의 갭 분석

#### 현재 설계 (expert_group_debate_v3.md 기준)
- ✅ AI 챗봇 1차 응답
- ✅ 에스컬레이션 (Slack 채널 생성)
- ✅ 담당자 연결
- ❌ **실시간 담당자 입장 표시 없음**
- ❌ **대기 중 진행 상황 메시지 없음**
- ❌ **구체적인 담당자 역할 분담 시나리오 없음**
- ❌ **단계별 응답 시나리오 (1차 안내 → 2차 답변 → SE 분석) 없음**
- ❌ **질문 분해 후 롤 자동 지정 로직 없음**

### 분석 2: 피드백의 핵심 의도

**고객 경험 관점**:
- 고객이 "누가 나를 도와주고 있는지" 실시간으로 알아야 함
- 고객이 "지금 무슨 일이 일어나고 있는지" 알아야 함
- 고객이 "언제쯤 답변을 받을 수 있는지" 예측 가능해야 함

**데모 임팩트 관점**:
- 단순 챗봇이 아닌 "실시간 협업 시스템"으로 보여야 함
- 여러 담당자가 동시에 움직이는 모습이 시각적으로 보여야 함
- "AI가 사람을 조율한다"는 인상을 줘야 함

**기술 구현 관점**:
- Agentic RAG의 "라우팅" 개념을 확장
- 질문 분해 → 각 서브질문에 담당 롤 매핑
- 실시간 상태 업데이트 UI

### 분석 3: 기존 요구사항과의 충돌/보완

#### 충돌 없음 (보완 관계)
- R4 (에스컬레이션): 피드백은 에스컬레이션 과정을 더 상세화
- R11 (AI 투명성): "AI 사고 과정"에 담당자 배정 과정 추가
- R12 (AI 코파일럿): 담당자별 역할에 맞는 답변 초안 제공

#### 새로운 요구사항 필요
- **R16 (신규)**: 실시간 담당자 상태 표시
- **R17 (신규)**: 단계별 진행 상황 메시지
- **R18 (신규)**: 질문 분해 후 롤 자동 배정

### Phase 2: 요구사항 업데이트 (신규 R16, R17, R18 추가)

#### R16: 실시간 담당자 상태 표시 (신규)
**Tier**: 2 (데모 임팩트 높음)
**설명**: 에스컬레이션 시 Slack 채널에 담당자가 입장하면 챗봇 UI에 실시간 표시
**인수 조건**:
1. Slack 채널 멤버 join 이벤트 감지
2. 챗봇 UI에 "OOO 담당자가 입장했습니다" 메시지 표시
3. 담당자 프로필 사진 + 역할 표시 (예: 송인찬 - 세일즈)
4. 최대 5명까지 표시 가능
**구현 시간**: 2시간 (Polling 방식) 또는 3시간 (WebSocket)

#### R17: 단계별 진행 상황 메시지 (신규)
**Tier**: 2 (고객 경험 핵심)
**설명**: AI 응답 지연 시 단계별 안내 메시지 자동 발송
**인수 조건**:
1. AI 응답 5초 초과 시 → "답변을 준비 중입니다" 메시지
2. 에스컬레이션 시작 시 → "담당자를 연결하고 있습니다" (채소희 역할)
3. 담당자 입장 시 → "OOO 담당자가 확인 중입니다"
4. 담당자 답변 준비 중 → "기술팀에 확인 중입니다" (송인찬 역할)
5. 최종 답변 전 → "곧 답변드리겠습니다" (이현진 역할)
**구현 시간**: 1.5시간 (메시지 템플릿 + 타이밍 로직)

#### R18: 질문 분해 후 롤 자동 배정 (신규)
**Tier**: 2 (에이전트화 핵심)
**설명**: CHECK 시맨틱 분석 결과를 기반으로 각 서브질문에 담당 롤 자동 배정
**인수 조건**:
1. CHECK 도메인 타입 → 담당 롤 매핑 테이블:
   - Product → 영업 (송인찬)
   - Feature → SE (이현진)
   - Process → 유지보수 (담당자 4)
   - Spec → 기술지원 (담당자 5)
2. 복합 질문 분해 시 각 서브질문에 담당 롤 자동 지정
3. 챗봇 UI에 "영업 담당자와 SE 담당자가 답변드릴 예정입니다" 메시지 표시
4. Slack 채널 생성 시 해당 롤의 담당자 자동 초대
**구현 시간**: 1시간 (CHECK 2단계에 통합)
