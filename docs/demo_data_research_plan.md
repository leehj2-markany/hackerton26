# 데모 데이터 리서치 플랜 (MCP 활용)

## 🎯 목적
실제 마크애니 고객 질문 패턴을 MCP 서버들로 리서치하여 데모 시나리오의 질문 데이터를 증강

---

## 📊 리서치 대상 MCP 서버

### 1. Jira MCP
**리서치 목표**: 고객사별 이슈 티켓에서 실제 질문 패턴 추출

**쿼리 계획**:
```jql
project = "고객지원" AND 
created >= -6M AND 
(labels = "SK하이닉스" OR labels = "국방부") AND
issuetype = "문의"
ORDER BY created DESC
```

**추출 데이터**:
- 질문 제목 (summary)
- 질문 내용 (description)
- 고객사명 (labels)
- 제품명 (components)
- 우선순위 (priority)
- 해결 시간 (resolution time)

**예상 결과**:
- SK하이닉스 관련 질문 20~30건
- 국방부 관련 질문 10~20건
- 질문 유형 분류: 기술/가격/절차/호환성

---

### 2. Confluence MCP
**리서치 목표**: 정책기능서, FAQ, 고객 사례에서 자주 묻는 질문 추출

**쿼리 계획**:
```
스페이스: "제품문서", "고객지원", "영업자료"
검색어: "FAQ", "자주 묻는 질문", "Q&A", "문의사항"
필터: 최근 1년 이내 업데이트
```

**추출 데이터**:
- FAQ 질문 목록
- 제품별 기술 문의 패턴
- 고객사별 특수 요구사항
- 보안 인증 관련 질문

**예상 결과**:
- 제품별 TOP 10 질문
- 업그레이드 관련 질문 패턴
- 호환성 관련 질문 패턴

---

### 3. Slack MCP
**리서치 목표**: hotline 채널에서 실시간 고객 질문 패턴 추출

**채널 리스트**:
- `squad-hotline-drm`
- `squad-hotline-dlp`
- `squad-hotline-tracer`
- `squad-hotline-ait-e-page`
- `0-markany-csteam`

**추출 데이터**:
- 최근 3개월 질문 메시지
- 질문 → 답변 쌍
- 에스컬레이션된 질문 (멘션 포함)
- 해결 시간 (첫 질문 → 최종 답변)

**예상 결과**:
- 실제 고객 질문 표현 방식
- 복합 질문 사례
- 긴급 문의 패턴

---

### 4. Google Drive MCP
**리서치 목표**: 제안서, 계약서, 기술문서에서 고객 요구사항 추출

**검색 계획**:
```
폴더: "제안서", "계약서", "기술지원"
파일명 키워드: "SK하이닉스", "국방부", "요구사항", "RFP"
파일 타입: PDF, DOCX
```

**추출 데이터**:
- 고객사별 특수 요구사항
- 제안서에 포함된 질문 리스트
- 기술 검토 요청 사항

**예상 결과**:
- SK하이닉스 제안서에서 실제 질문 5~10건
- 국방부 RFP에서 보안 요구사항 질문

---

### 5. Gmail MCP (선택)
**리서치 목표**: 고객 이메일에서 문의 패턴 추출

**검색 계획**:
```
from: @skhynix.com OR @mnd.go.kr
subject: 문의 OR 질문 OR 요청
date: 최근 6개월
```

**추출 데이터**:
- 이메일 제목 (질문 요약)
- 이메일 본문 (상세 질문)
- 답변 이메일 (해결 방법)

## 🔍 리서치 실행 계획

### Phase 1: Jira 리서치 (30분)

3. 고객 사례 문서에서 질문 패턴 추출

### Phase 3: Slack 리서치 (30분)

2. 질문 → 답변 쌍 추출
3. 복합 질문 사례 선별
4. 실제 표현 방식 학습

### Phase 4: Google Drive 리서치 (30분)
1. SK하이닉스 제안서 검색
2. 국방부 RFP 문서 검색
3. 요구사항 섹션에서 질문 추출

### Phase 5: 데이터 통합 & 증강 (30분)
1. 리서치 결과 통합
2. 데모 시나리오별 질문 선정
3. 질문 표현 자연스럽게 다듬기
4. 답변 데이터 준비
