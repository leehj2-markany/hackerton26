## 🔴 Critical Issues (테스트에서 발견)

### Issue 1: Slack 채널 답변이 챗봇으로 전달되지 않음
- **재현 경로**: 에스컬레이션 → Slack 채널 생성 → 담당자가 채널에서 답변 → 챗봇에 표시 안 됨
- **스크린샷**: Slack `#esc-drm-0312-0531` 채널에서 이현진(SE)이 답변했으나 챗봇 UI에 미반영
- **예상 원인**: 
  1. `setSlackPollSince` 함수가 미정의 (state → ref 마이그레이션 시 누락)
  2. 폴링 타이밍 이슈 — 에스컬레이션 시퀀스 완료 후 백그라운드 폴링이 동작하지 않을 수 있음
  3. `slackChannelId`가 올바르게 설정되지 않아 폴링 대상 채널이 다를 수 있음
- **영향도**: Critical — 핵심 기능(담당자 실시간 답변) 불가
- **상태**: ✅ 수정 완료
  - pollStartTs를 2분 전으로 앞당겨 시퀀스 중 답변 누락 방지
  - 가상/실제 에이전트 순차 처리 → 병렬 처리로 구조 변경
  - 단일 폴링 루프에서 모든 실제 에이전트를 동시 감시

### Issue 2: 담당자 답변 대기 중 채팅창 비활성화
- **재현 경로**: 에스컬레이션 → "담당자가 답변을 준비하고 있습니다..." 표시 → 입력창 비활성화
- **스크린샷**: 입력창에 "담당자 답변 대기 중..." placeholder + 전송 버튼 비활성화
- **예상 원인**: `isEscalationBusy` 상태가 폴링 완료 전까지 true로 유지되어 입력 차단
- **영향도**: High — 사용자가 대기 중 추가 질문/취소 불가
- **기대 동작**: 담당자 답변 대기 중에도 사용자가 추가 질문을 입력할 수 있어야 함
- **상태**: ✅ 수정 완료
  - 입력 disabled 조건을 `isAIProcessing || isEscalationBusy`로 분리 (폴링 중에는 둘 다 false)
  - 에스컬레이션 초기 시퀀스(채소희 입장~Slack 전송)만 busy, 폴링 구간에서는 입력 활성화
  - placeholder를 상태별로 세분화: "AI 답변 생성 중..." / "담당자 연결 중..." / "담당자에게 질문하기..."

## 🟡 기존 UI/UX 개선 포인트 (코드 리뷰 기반)

### Issue 3: Chatbot.jsx 단일 파일 비대화 (1300+ lines)
- 모든 로직(intake, 메시지, 에스컬레이션, 폴링)이 하나의 컴포넌트에 집중
- 커스텀 훅 분리 권장: `useSlackPolling`, `useEscalation`, `useAIChat`

### Issue 4: 에스컬레이션 버튼 이중 표시 가능성
- AI 답변에 `showEscalation: true`가 설정된 후, 사용자가 추가 질문 시 이전 버튼 비활성화 처리는 되어 있음
- 단, 빠른 연속 클릭 시 중복 에스컬레이션 요청 가능 (debounce 없음)

### Issue 5: 모바일 반응형 미흡
- Homepage 헤더 네비게이션이 `hidden md:flex`로 모바일에서 숨겨지지만 햄버거 메뉴 없음
- 챗봇 위젯이 모바일에서 전체 화면을 차지하지 않아 사용성 저하 가능

### Issue 6: 세션 복원 시 에스컬레이션 상태 미복원
- `sessionStorage`에서 고객 정보와 채널 ID는 복원하지만, `escalationMode` 상태는 복원하지 않음
- 페이지 새로고침 시 에스컬레이션 모드가 초기화되어 AI 모드로 돌아감

### Issue 7: ThinkingPanel 애니메이션 타이밍
- 각 단계가 300ms 간격으로 표시되는데, 실제 LLM 응답이 빠르면 ThinkingPanel이 닫히기 전에 답변이 표시될 수 있음
- 최소 표시 시간 보장 로직 없음
